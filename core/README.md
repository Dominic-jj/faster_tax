# core/ — 核心模块

## 模块概览

| 模块 | 职责 |
|------|------|
| `ocr_service.py` | OCR 文字识别（多引擎：EasyOCR / RapidOCR / PaddleOCR） |
| `parser.py` | 从 OCR 文本行中提取结构化物品数据 |
| `tax_matcher.py` | 物品名 → 税收分类编码的智能匹配 |

## 税收分类匹配算法（tax_matcher.py）

### 数据源

加载 `data/tax_list.xlsx`（4200+ 条税收分类记录），每条包含：

- **合并编码**（L 列）— 税收分类编码
- **名称**（M 列）— 分类名称
- **分类名**（N 列）— 所属分类
- **说明**（O 列）— 商品说明/范围描述

### 匹配流程

#### 第一步：关键词提取

从物品名中提取中文子串作为匹配关键词，同时生成 2-gram 分词和尾部产品类型词。

示例：`"16A 空气开关"` → `["空气开关", "空气", "开关", "气开"]`

提取尾部 1-3 字作为产品类型关键词（如 `"开关"`、`"气开关"`），这类词代表物品的核心功能，匹配时权重更高。

#### 第二步：逐条评分

对 4200+ 条分类记录逐一计算匹配分数：

| 匹配类型 | 加分规则 | 说明 |
|---------|---------|------|
| 完整物品名命中分类名称 | `len(物品名) × 10` | 最高优先级，几乎直接命中 |
| 完整物品名命中说明字段 | `len(物品名) × 5` | 次高优先级 |
| 单个关键词命中分类名称 | `len(关键词) × 3` + 尾部加权 | 核心评分逻辑 |
| 关键词仅命中说明（未命中名称） | 权重 × 0.3 | 大幅降权，避免"说明里提到但分类不对"的误匹配 |
| 多个关键词同时命中同一分类 | 分数 × 1.5 | 语义一致性奖励，多个词都指向同一分类说明匹配更准确 |

**尾部产品类型词加权**（物品名末尾 1-3 字，代表核心功能）：

| 位置 | 额外加权 |
|------|---------|
| 最后 1 字 | +10 |
| 最后 2 字 | +6 |
| 最后 3 字 | +4 |

#### 第三步：同分排序偏好

分数相同时的分类优先级：

1. **机电设备**（`109` 开头）→ +3
2. **非金属矿物**（`108` 开头）→ +2
3. **其他商品**（`1` 开头）→ +2
4. **短名称优先** — 名称越短越通用，略优（`-len(name) × 0.05`）

### 算法本质

**基于关键词的加权模糊匹配，不是机器学习模型。**

核心思路是计算物品名与分类名称/说明的文字重叠度，通过多层加权（完整匹配 > 部分匹配 > 说明匹配）和产品类型词加权，选出得分最高的税收分类编码。

### 示例

```
输入: "空气开关"
关键词: ["空气开关", "空气", "开关", "气开"]
匹配结果: 109040000 → 开关设备

输入: "平板灯"
关键词: ["平板灯", "平板", "板灯", "灯"]
匹配结果: 109040000 → 照明器具

输入: "电线"
关键词: ["电线"]
匹配结果: 109040000 → 电线电缆
```
